## 발표 관련

### 시간 관리
- **발표 시간 준수**: 시간이 오바되면 준비 미흡으로 판단됨

- 그래프와 차트 작성 필수사항:
  - 제목, 축 이름 반드시 포함
  - 단위 표기 (매우 중요!)
- PPT 슬라이드에 그림만 넣지 말 것
  - 최소 1줄 이상의 설명 텍스트 추가

## 스크립트 작성법

### 개요
매번 환경설정을 쉘별로 일일이 하지 말고, 스크립트로 자동화할 것

### 기본 템플릿

**1. 스크립트 파일 생성**
```bash
vim [스크립트이름].sh
```

**2. 내용 작성**
```bash
#!/bin/bash

source /opt/ros/humble/setup.bash

sleep 1

set -o errexit
set -o nounset
set -o pipefail

# 스크립트 내용 추가

exit
```

**3. 실행 권한 부여 및 실행**
```bash
sudo chmod +x [스크립트이름].sh
./[스크립트이름].sh
```

### 참고
- Bash style guide 참고
- 더 자세한 내용은 Gemini에 질문

## ⌨️ Vim 사용법

### .vimrc 설정 (필수!)

**1. vim 설정 파일 열기**
```bash
vi ~/.vimrc
```

**2. 루트 사용자도 사용할 수 있도록 복사**
```bash
sudo cp ~/.vimrc /root/.vimrc
```

**3. 설정 예시**
```bash
vi s.sh
sudo chmod +x s.sh
./s.sh
```
**4. 소싱이 안된다면**
```bash
source source.sh
```
./s.sh는 사이드 터미널을 열어 해당 터미널에서 실행하고 끄는 방식

source s.sh는 해당 터미널에 바로 소싱을 진행하는 방식

## 터미널 도구 

# Terminator vs tmux 비교 분석

## 1. Terminator (GUI Terminal Emulator)
Terminator는 리눅스 데스크탑 환경에서 실행되는 **창 관리 특화 터미널**입니다. 마우스 조작이 직관적이며 화면 분할이 매우 쉽습니다.

* **정체성:** 독립적인 터미널 프로그램 (애플리케이션).
* **장점:** - 마우스 우클릭으로 간편하게 화면 분할 (수평/수직).
    - 드래그 앤 드롭으로 창 크기 조절 가능.
    - 직관적인 GUI 설정 화면 제공.
* **단점:** GUI 환경(X-Window)이 반드시 필요하며, 창을 닫으면 실행 중인 프로세스가 종료됨.

---

## 2. tmux (Terminal Multiplexer)
tmux는 터미널 내부에서 실행되는 **세션 관리 도구**입니다. 텍스트 기반으로 작동하며 서버 작업에 필수적입니다.

* **정체성:** 터미널 내에서 돌아가는 유틸리티 (CLI 도구).
* **장점:** - **세션 유지:** 접속이 끊겨도 서버에서 작업이 계속 실행됨 (Detach/Attach).
    - **이동성:** SSH로 원격 서버에 접속해도 로컬과 동일한 환경 사용 가능.
    - 모든 기능을 키보드 단축키로 제어 (숙련 시 속도 비약적 상승).
* **단점:** 초기 학습 곡선이 있고, 기본 설정에서는 마우스 사용이 제한적임.

---

## 3. 주요 차이점 비교표

| 구분 | Terminator | tmux |
| :--- | :--- | :--- |
| **작동 환경** | GUI (리눅스 데스크탑) | CLI (터미널, SSH, 로컬 어디든) |
| **화면 분할** | 마우스 + 키보드 (매우 쉬움) | 키보드 단축키 중심 (Prefix 활용) |
| **세션 관리** | 창 닫으면 종료 | 창 닫아도 백그라운드 유지 가능 |
| **설정 방식** | GUI 설정창 또는 설정 파일 | `.tmux.conf` 텍스트 파일 수정 |
| **추천 대상** | 로컬 리눅스 사용자 | 서버 관리자, 개발자, SSH 사용자 |

---

## 4. 요약 가이드
- **Terminator:** "나는 리눅스 화면에서 마우스로 편하게 창을 쪼개서 쓰고 싶다."
- **tmux:** "나는 서버 작업이 많고, 접속이 끊겨도 내 작업이 유지되길 원한다."
